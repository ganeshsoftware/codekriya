Encapsulation:
In object-oriented computer programming (OOP) languages, the notion of encapsulation (or OOP Encapsulation) refers to the bundling of data, along with the methods that operate on that data, into a single unit. Many programming languages use encapsulation frequently in the form of classes.
Abstraction:
Abstraction is commonly defined as the extraction of relevant information from a larger data set, where utilizing abstraction allows engineers and others to simplify a codebase.
Inheritance:
Inheritance is the procedure in which one class inherits the attributes and methods of another class. The class whose properties and methods are inherited is known as the Parent class. And the class that inherits the properties from the parent class is the Child class.

Fig.1 The .NET 2-Step compilation process.
  1. A set of compilers (C# compiler, VB.NET compiler, VC++.NET compiler, J# compiler etc.)
  2. The Just In Time (JIT) compiler
  3. The Common Language Runtime (CLR)
  4. The MSIL (Microsoft Intermediate Language) formats.
  5. Framework service components like Garbage Collector (GC), Process Optimizer etc.
Writing code for the .NET Framework using languages like C# and getting the output as MSIL, and then further compiling it to the native code (called as 2-STEP compilation) has ONE BIG Advantage - Platform Independence.
